- 关联：[ROADMAP.md](ROADMAP.md) / [ARCHITECTURE.md](ARCHITECTURE.md) / [REVIEW.md](REVIEW.md)

---

## 背景

当前"设置"**散落在账户管理页**（「后台刷新全部账户」开关 + 「额度告警」开关），没有独立设置入口。用户调设置需先进账户页，不合直觉。

更关键：**系统引导缺失**——电池优化白名单、通知权限这些是解决已知痛点（华为 widget 后台不刷新、通知不响）的产品化正解，目前没有出口让用户配置/修复。

## 设计目标

集中所有用户可配置项、一站式；分组清晰；默认值合理（多数用户不改）；重点解决真痛点（系统层限制的用户引导）。

---

## 结构：独立 SettingsScreen

入口：用量页/账户页右上角「⚙️ 设置」。按用户心智分 6 组：

| 组 | 项 | 现状/必要性 |
|---|---|---|
| **刷新** | 后台刷新范围（仅 active / 全部） | ⚠️ 已有，从账户页搬过来 |
| **告警** | 告警开关 | ⚠️ 已有，搬过来；未来加阈值/免打扰时段 |
| **系统引导** | 通知权限状态+跳转、**电池优化白名单+跳转**、启动管理引导 | 🔴 **最高价值——见下** |
| **数据** | 清除缓存、清除趋势历史、清除全部账户 | 🟡 趋势功能后数据会堆积，需清理出口 |
| **关于** | 版本号、合规声明、ToS、GitHub 链接、反馈 | 🟡 发布前要有（合规 + 侵权声明） |
| **显示**（可选） | 主题、余量/用量切换 | 🟢 低优先，余量/用量之前评估不做 |

---

## 关键洞察：系统引导组是最高 ROI

widget 半小时不刷新（用户实测）诊断结论：**华为/HarmonyOS 省电限制冻结 WorkManager**，代码绕不过，只能靠用户手动开：
- **电池优化白名单**（`REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` → 弹系统设置）
- **启动管理**（跳华为「应用启动管理」→ 三开关全开：自启动/关联启动/后台活动）

设置页放一个「**后台刷新不工作？点这里修复**」引导，直接指向这两个系统设置——把"用户遇到的坑"产品化，比任何代码优化都管用（因为是系统层限制）。通知权限同理（Android 13+ 用户可能误拒）。

> 这是当前 widget 后台刷新体验的真正瓶颈。Worker 代码链已验证正常（dumpsys 实证 scheduled + 未被限制），但华为系统层会延迟/冻结，用户不开白名单就时灵时不灵。

---

## MVP（第一版）

1. 独立 `SettingsScreen` + 入口（用量页右上 ⚙️）
2. 把现有的「后台刷新 + 告警」两个开关搬过来（账户页只留账户相关操作）
3. **系统引导组**：通知权限 + 电池白名单两个跳转按钮（带「不工作点这里」文案）
4. 关于区：版本 + 合规声明 + GitHub 链接

## 后续（YAGNI，等真需求）

- 告警阈值滑块 / 免打扰时段
- 主题（深/浅）
- 余量/用量切换（[prototype 评估](../prototype/display-mode.html) 后决定不做，可重审）
- 手动刷新频率

## 不做

- 设置项堆砌——每项要有真需求，避免稀释。早期产品克制优于堆选项。
